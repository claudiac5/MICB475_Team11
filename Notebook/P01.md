
# P01 - QIIME 2 Demultiplexing 

Feb. 

## Purpose:
To import raw 16S rRNA sequencing reads into QIIME 2 and perform demultiplexing to assign each sequence read to its corresponding sample using barcode information.

## Material: 
- Sample metadata file 
- QIIME 2 environment installed on the server
- Access to the remote server via VPN
- Terminal interface for running shell commands
- QIIME 2 Viewer

## Method:
1. Logged into the remote server through VPN.
2. Imported raw sequencing reads into QIIME 2.
3. Generated a QIIME artifact file containing the demultiplexed sequences.
4. Created a visualization file.
5. Transferred the visualization file from the server to the local computer using scp.
6. Uploaded the .qzv file to the QIIME 2 Viewer website.

## Results: 
The demux.qzv visualization provided a summary of sequencing reads across all samples. The interactive quality plot showed how sequence quality varied across read positions, which can help determine appropriate trimming parameters for the denoising step.

## Discussion:
Demultiplexing assigned sequencing reads to their corresponding samples using barcode information. The  visualization confirmed that reads were distributed across samples and allowed for inspection of sequencing quality.

Ensuring high-quality reads is critical, as poor-quality bases can introduce sequencing errors and impact downstream identification of amplicon sequence variants (ASVs).

## Future direction:
The next step is denoising the demultiplexed sequences using DADA2 to remove low-quality reads and trim sequences to a consistent length.
